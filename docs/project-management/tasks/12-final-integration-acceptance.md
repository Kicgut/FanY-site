# Phase 12：最终集成与上线验收

## 1. 任务目标

在所有阶段完成后，进行端到端验收，确保公开网站、照片权限、Hermes 对话、内容流水线、Skill Registry、备份告警和本地最高权限模型整体可用。

## 2. 前置条件

- Phase 1-11 已完成或明确跳过。
- 所有未确认决策已记录。
- 已有数据库和文件备份。

## 3. 验收范围

### 公开访客

- 首页可访问。
- 博客可访问。
- 公开摄影作品可访问。
- 不能访问 `/ai`。
- 不能访问朋友/私人照片。

### 登录朋友

- 能登录。
- 只能访问授权照片。
- 可上传照片。
- 上传后进入 pending。
- 可查看自己上传内容的审核状态。
- 无权访问后台。
- 无权删除照片。

### 授权 AI 用户

- 能访问 `/ai`。
- 只能普通问答或限定能力。
- 不能执行 shell。
- 不能控制设备。

### 你本人远程

- 能管理用户。
- 能审核照片。
- 能查看私人照片，需强认证或至少权限确认。
- 能查看 Hermes 整理结果。
- 能查看 Skill Registry。
- 不能永久删除或执行高危操作。

### 你本人本地

- 能执行本地最高权限操作。
- 所有高危操作有审计日志。

## 4. 必测场景

1. 朋友上传照片 → pending → 你批准为 family → 只有 family 可见。
2. 朋友上传照片 → rejected → 不展示。
3. public 照片缩略图同步到 ECS。
4. friends/private 照片未同步到 ECS 公开目录。
5. 未授权用户访问 `/ai` 返回 403。
6. 授权用户访问 `/ai` 成功。
7. 通过 `/ai` 请求执行命令失败。
8. Hermes 生成候选博客后不会自动公开。
9. Skill 新增后显示为 new/pending。
10. 远程 owner 调用永久删除 API 返回 403。
11. local_trusted owner 调用高危 API 需二次确认并记录日志。
12. 数据库备份可恢复到测试副本。

## 5. 上线前检查

- `.env` 未提交 Git。
- JWT_SECRET 足够强。
- frp token 足够强。
- Nginx 未公开 Hermes WebUI。
- 本地 photo-server 不直接暴露公网。
- 所有 admin API 有服务端权限校验。
- 所有上传接口限制大小、类型、配额。
- 所有文件路径使用安全解析。
- 备份任务已运行至少一次。

## 6. 验收输出

生成：

```text
docs/agent-context/verification/final-acceptance-YYYY-MM-DD.md
```

包含：

- 测试人。
- 测试时间。
- 版本/commit。
- 通过项。
- 失败项。
- 风险项。
- 是否允许上线。

## 7. 不通过时处理

- 安全项失败：禁止上线。
- 数据丢失风险：禁止上线。
- UI 小问题：可记录后上线。
- Hermes 安全边界失败：禁止上线 `/ai` 和 `/admin/hermes`。

## 8. Agent 注意事项

- 不允许用“看起来能用”代替验收表。
- 不允许跳过权限测试。
