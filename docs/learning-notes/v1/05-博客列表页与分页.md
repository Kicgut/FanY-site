## 核心概念：computed 派生状态

**类比：** computed 就像 Excel 公式。A1=10, B1=20, C1=A1+B1 → C1 自动显示 30。当 A1 改为 15，C1 自动变成 35。

```typescript
const allTags = computed(() => { ... })      // 从文章列表提取所有标签
const filteredPosts = computed(() => { ... }) // 根据选中标签过滤
const paginatedPosts = computed(() => { ... }) // 从过滤结果分页
```

**数据流向：** posts → allTags(标签池) → filteredPosts(过滤) → paginatedPosts(分页) → 页面渲染

## 对比：前端分页 vs 后端分页

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **前端分页** | 一次加载全部，JS 切片 | 简单、切换快 | 数据量大时慢 |
| 后端分页 | 每次请求一页数据 | 省流量、支持大数据 | 每次翻页要请求 |
| 无限滚动 | 滚动到底加载更多 | 体验好 | 实现复杂 |
