## 核心概念：Layout 布局系统

Layout 就像相框。你有 10 幅画（页面），每幅画内容不同，但都用同一个相框（导航+页脚）。

```
┌─────────────────────────┐
│  Nav（仅在 layout 写一次）│ ← layouts/default.vue
├─────────────────────────┤
│  <slot /> = 页面内容     │ ← pages/*.vue 放这里
├─────────────────────────┤
│  Footer（仅写一次）      │
└─────────────────────────┘
```

**关系链：** app.vue → NuxtLayout → layouts/default.vue → slot → pages/*.vue

## 对比：CSS 变量 vs 其他方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **CSS 变量** | 运行时可改、零依赖、Element Plus 兼容 | 嵌套需手写 |
| SCSS/Less | 变量+嵌套+混入 | 运行时不可改、需编译器 |
| Tailwind | 开发快 | HTML 很长 |

## 人格标签
> 空间设计师 — 理解布局骨架，把页面拆成外壳(layout)和内容(page)两层

## 核心概念：前端路由

传统网站 = 每换一页重新盖楼。SPA = 同一栋楼，只换房间家具。

**文件即路由：**
```
pages/
├── index.vue          → /
├── about.vue          → /about
├── blog/
│   ├── index.vue      → /blog
│   └── [id].vue       → /blog/123 ← 动态路由
└── portfolio/
    └── index.vue      → /portfolio
```

## 对比：导航方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **NuxtLink** | 自动预加载、无刷新 | 仅限 Nuxt |
| `<a>` 标签 | 简单 | 整页刷新白屏 |
| useRouter().push() | 可在 JS 中触发 | 不如声明式 |
