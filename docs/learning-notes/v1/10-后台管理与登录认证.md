## 核心概念：认证 vs 授权

**认证（Authentication）：** "你是谁？" → 登录验证身份
**授权（Authorization）：** "你能做什么？" → 检查权限（admin vs user）

**JWT 工作流程：**
```
用户输入密码 → 服务端验证 → 生成 JWT Token → 返回给前端
前端存储 Token → 每次请求带 Authorization: Bearer <token>
服务端验证 Token → 解析出用户信息 → 放行或拒绝
```

**JWT 结构：** `Header.Payload.Signature`（三段用`.`分隔的 Base64）
- Header: 算法类型
- Payload: 用户数据（id, role, 过期时间）
- Signature: 签名（防篡改）

## 对比：认证方案

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **JWT Token** | 无状态令牌 | 无状态、可扩展 | 无法主动失效 |
| Session + Cookie | 服务端存储 | 可主动失效 | 需要共享存储 |
| OAuth2 | 第三方授权 | 社交登录 | 复杂 |
| Passkey/WebAuthn | 无密码 | 最安全 | 浏览器兼容性 |
