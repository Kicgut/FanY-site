## 📖 核心概念：数据库迁移

**类比：** 数据库迁移就像房子装修——你不能拆了重建，只能一步步改造，而且要保证住在里面的人不受影响。

### 什么是迁移？

```
迁移前的 User 表：
┌────┬──────────┬──────────┬──────┐
│ id │ username │ password │ role │
└────┴──────────┴──────────┴──────┘

迁移后的 User 表：
┌────┬──────────┬──────────┬──────┬─────────┬───────────┬──────────────┐
│ id │ username │ password │ role │ groups  │ status    │ aiAccessLevel│
└────┴──────────┴──────────┴──────┴─────────┴───────────┴──────────────┘
         新增 ↑         新增 ↑       新增 ↑        新增 ↑
```

### 为什么不能直接改表结构？

```
❌ 直接改：ALTER TABLE User ADD COLUMN groups TEXT
   问题：
   1. 其他开发者不知道你改了
   2. 生产环境和开发环境不一致
   3. 出错了无法回滚
   4. 新加入的开发者不知道数据库长什么样

✅ 用迁移：
   1. 修改 schema.prisma（声明"我想要什么"）
   2. npx prisma migrate dev（自动生成 SQL）
   3. 迁移文件提交到 Git（所有人都能看到）
   4. 生产环境 npx prisma deploy（自动执行）
```

## 💻 代码解读

### Prisma Schema 新增字段

```prisma
model User {
  // 原有字段...
  role          String   @default("admin")
  
  // Phase 2 新增字段 ↓
  groups        String?                    // ← ? 表示可选（可以是 null）
  // 存储格式：JSON 字符串 '["family","close-friends"]'
  // 为什么用 String 而不是数组？SQLite 不支持数组类型
  
  status        String   @default("active")
  // 用户状态：active / disabled / pending
  // @default 表示新建用户默认是 active
  
  aiAccess      Boolean  @default(false)
  // 是否允许使用 AI 功能
  
  aiAccessLevel String   @default("none")
  // AI 访问等级：none / chat / limited / owner
  
  lastLoginAt   DateTime?
  // 最后登录时间，用于安全审计
  
  uploadQuotaMb Int      @default(500)
  // 上传配额，单位 MB
  
  usedQuotaMb   Int      @default(0)
  // 已使用配额
}
```

### Photo 表新增字段

```prisma
model Photo {
  // 原有字段...
  
  // Phase 2 新增 ↓
  visibility    String   @default("public")
  // 可见性：public / friends / private
  
  visibleTo     String?  // 可见的朋友 ID 列表（JSON）
  // 当 visibility=friends 时，指定哪些朋友可见
  // 格式：'[1,2,3]'（用户 ID 列表）
  
  status        String   @default("active")
  // 照片状态：active / archived / deleted
  
  reviewStatus  String   @default("pending")
  // 审核状态：pending / approved / rejected
  // 朋友上传的照片需要审核
  
  reviewedAt    DateTime?
  // 审核时间
}
```

### 新模型：AuditLog（审计日志）

```prisma
model AuditLog {
  id        Int      @id @default(autoincrement())
  userId    Int?                  // 操作者（系统操作时为 null）
  action    String                // 操作类型：login / upload / delete / approve...
  target    String?               // 操作对象：photo:123 / user:456
  detail    String?               // 详细信息（JSON）
  ip        String?               // 操作者 IP
  createdAt DateTime @default(now())
}
```

**为什么要审计日志？**
```
场景：朋友上传了一张不合适的照片，你批准发布了
问题：谁批准的？什么时候？为什么？

有审计日志：
  [2026-07-04 15:30] user=admin action=approve photo=123 ip=192.168.1.100
  → 清晰可追溯

没审计日志：
  → 只能猜，或者翻数据库记录
```

## 🔍 对比与选型

### 数据库迁移工具对比

| 工具 | 语言 | 优点 | 缺点 | 适合 |
|------|------|------|------|------|
| Prisma Migrate | TypeScript | 自动生成 SQL、类型安全 | 仅支持 Prisma | Node.js 项目 |
| Flyway | Java | 业界标准、支持多数据库 | 配置复杂 | Java 项目 |
| Alembic | Python | SQLAlchemy 生态 | 需手写迁移 | Python 项目 |
| Knex.js | JavaScript | 轻量、灵活 | 没有类型安全 | 轻量 Node 项目 |

**为什么选 Prisma Migrate：** 本项目用 Nuxt 3 + TypeScript，Prisma 是 Nuxt 官方推荐的 ORM，迁移文件自动生成，不需要手写 SQL。

### 数据库选择对比

| 数据库 | 类型 | 优点 | 缺点 | 适合 |
|--------|------|------|------|------|
| SQLite | 文件型 | 零配置、单文件备份 | 不支持并发写 | 小项目、个人网站 |
| PostgreSQL | 服务型 | 功能最强、支持并发 | 需要单独安装维护 | 中大型项目 |
| MySQL | 服务型 | 生态成熟 | 功能不如 PG | 传统 Web 项目 |
| MongoDB | 文档型 | 灵活 schema | 查询不如 SQL 直觉 | 内容管理系统 |

**为什么选 SQLite：** 个人网站访问量小，SQLite 单文件备份简单，Docker 容器内直接用文件数据库，不需要额外安装数据库服务。

### `as const` vs 枚举 vs 字符串

```typescript
// 方式 1：as const（本项目选择）
const STATUS = { ACTIVE: 'active', DISABLED: 'disabled' } as const
// 优点：运行时是普通对象，编译时有精确类型
// 编译后：{ ACTIVE: 'active', DISABLED: 'disabled' }

// 方式 2：TypeScript enum
enum Status { ACTIVE = 'active', DISABLED = 'disabled' }
// 优点：语法简洁
// 缺点：编译后生成额外代码，tree-shaking 不友好

// 方式 3：字符串字面量
type Status = 'active' | 'disabled'
// 优点：最轻量
// 缺点：没有运行时的键名映射（如 STATUS.ACTIVE）
```
