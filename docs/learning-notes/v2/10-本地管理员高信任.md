## 📖 核心概念：信任边界

**类比：** 就像银行——柜台办理（本地）可以做所有业务，ATM（远程）只能取款转账，不能修改账户信息。

### 信任等级

```
最高信任 ──────────────────────────────── 最低信任
    │                                        │
本地管理员                              公开访客
(同一内网)                              (互联网)
    │                                        │
可以：                                  可以：
  ✅ 查看所有内容                           ✅ 查看公开内容
  ✅ 上传/删除照片                         ❌ 不能上传
  ✅ 执行 Shell 命令                       ❌ 不能管理
  ✅ 修改系统配置                           ❌ 不能修改
  ✅ 批量操作                               ❌ 不能批量
```

### 为什么远程不能删除？

```
场景：你在外面用手机登录后台，想删除一张照片

风险：
1. 手机可能被盗
2. 公共 WiFi 可能被监听
3. 远程 IP 可能被伪造

安全原则：
  删除操作不可逆 → 只在可信环境（本地）执行
  远程只能"标记删除"，回到本地再真正删除
```

## 💻 代码解读

### 来源判定函数

```typescript
// server/utils/permission.ts

export type AccessOrigin = 'local_trusted' | 'remote_owner' | 'remote_user' | 'public'

// 可信 CIDR 列表
const TRUSTED_CIDRS = [
  '127.0.0.0/8',      // 本地回环
  '10.0.0.0/8',       // 内网 A 类
  '172.16.0.0/12',    // 内网 B 类
  '192.168.0.0/16',   // 内网 C 类
]

export function getAccessOrigin(event: H3Event): AccessOrigin {
  // 获取真实 IP（考虑反向代理）
  const ip = getRequestHeader(event, 'x-forwarded-for') || 
             getRequestHeader(event, 'x-real-ip') || 
             '127.0.0.1'
  
  // 1. 检查是否是本地可信 IP
  for (const cidr of TRUSTED_CIDRS) {
    if (isInCidr(ip, cidr)) {
      return 'local_trusted'
    }
  }
  
  // 2. 检查是否是已登录的管理员
  const user = event.context.authUser
  if (user?.role === 'admin') {
    return 'remote_owner'
  }
  
  // 3. 检查是否是已登录的朋友
  if (user?.role === 'friend') {
    return 'remote_user'
  }
  
  // 4. 未登录
  return 'public'
}
```

**关键点：CIDR 匹配**
```typescript
function isInCidr(ip: string, cidr: string): boolean {
  const [base, prefixLen] = cidr.split('/')
  const ipNum = ipToNumber(ip)
  const baseNum = ipToNumber(base)
  const mask = ~((1 << (32 - Number(prefixLen))) - 1)
  
  return (ipNum & mask) === (baseNum & mask)
}

// 192.168.1.100 在 192.168.0.0/16 内吗？
// /16 表示前 16 位是网络号
// 192.168.0.0   → 11000000.10101000.00000000.00000000
// 192.168.1.100 → 11000000.10101000.00000001.01100100
// 前 16 位相同 → 在同一网段 ✅
```

### 权限检查组合函数

```typescript
export function requireLocalAccess(event: H3Event): boolean {
  return getAccessOrigin(event) === 'local_trusted'
}

export function requireAdmin(user: AuthUser): boolean {
  return user.role === ROLES.ADMIN && user.status === STATUS.ACTIVE
}

// 使用示例
export default defineEventHandler(async (event) => {
  const user = event.context.authUser
  
  // 必须是管理员
  if (!user || !requireAdmin(user)) {
    throw createError({ statusCode: 403, message: '需要管理员权限' })
  }
  
  // 删除操作必须本地
  if (!requireLocalAccess(event)) {
    throw createError({ statusCode: 403, message: '删除操作只允许本地执行' })
  }
  
  // 执行删除...
})
```

## 🔍 对比与选型

### 权限模型对比

| 模型 | 原理 | 优点 | 缺点 | 适合 |
|------|------|------|------|------|
| RBAC | 基于角色 | 简单直观 | 无法区分来源 | 传统项目 |
| ABAC | 基于属性 | 灵活 | 复杂 | 企业项目 |
| 本项目 | RBAC + 来源 | 平衡安全和简单 | 需要自定义 | 个人项目 |

**本项目的三维权限模型：**
```
权限 = f(身份, 来源, 操作)

示例：
  admin + local_trusted + delete = ✅ 允许
  admin + remote_owner + delete = ❌ 拒绝
  friend + remote_user + view(private) = ❌ 拒绝
  viewer + public + view(public) = ✅ 允许
```

### IP 来源判定方案对比

| 方式 | 优点 | 缺点 | 适合 |
|------|------|------|------|
| CIDR 匹配 | 精确、灵活 | 需要理解网络 | 本项目 |
| 简单字符串比较 | 最简单 | 不支持网段 | 硬编码 IP |
| GeoIP 数据库 | 可以按地区 | 需要更新、有误差 | 按地区限制 |
| VPN 检测 | 可以识别 VPN | 不可靠 | 高安全需求 |

**为什么选 CIDR：** 支持整个内网网段判定，不需要列出每个 IP，适合动态内网环境。

### 双因素认证方案对比

| 方案 | 安全性 | 用户体验 | 成本 | 适合 |
|------|--------|---------|------|------|
| 无 2FA | 低 | 最好 | 无 | 低风险 |
| SMS 验证码 | 中 | 好 | 有 | 一般应用 |
| TOTP（Google Authenticator） | 高 | 中 | 无 | 推荐 |
| 硬件密钥（YubiKey） | 最高 | 差 | 贵 | 高安全 |

**本项目暂不实现 2FA：** 个人网站，访问量小，本地信任模型已经足够。未来可以加 TOTP。
