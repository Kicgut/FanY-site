## 📖 核心概念：内容流水线

**类比：** 内容流水线就像工厂生产线——原材料（AI 生成/手动编写）→ 质检（审核）→ 包装（格式化）→ 上架（发布）。

### 流水线阶段

```
AI 生成/手动编写
      ↓
┌─────────────┐
│   draft     │ ← 草稿，可编辑
│   (草稿)    │
└──────┬──────┘
       │ 提交审核
       ▼
┌─────────────┐
│   pending   │ ← 等待审核
│   (待审核)   │
└──────┬──────┘
       │
  ┌────┴────┐
  ↓         ↓
┌─────┐  ┌──────┐
│reject│  │approve│
│(拒绝)│  │(批准) │
└─────┘  └───┬──┘
              │ 发布
              ▼
        ┌──────────┐
        │ published │ ← 出现在网站上
        │ (已发布)  │
        └──────────┘
```

## 💻 代码解读

### 内容候选模型

```typescript
// server/services/content-pipeline.ts

export interface CandidateMeta {
  id: string            // UUID
  title: string         // 标题
  source: string        // 来源：ai_chat / manual / import
  status: string        // 状态：draft / pending / approved / rejected
  tags: string[]        // 标签
  description?: string  // 描述
  createdAt: string     // 创建时间
  updatedAt: string     // 更新时间
  rejectReason?: string // 拒绝原因
}

export interface CandidateInput {
  title: string
  content: string       // Markdown 内容
  description?: string
  tags?: string[]
}
```

### 存储结构

```
/mnt/data/personal-website/content-pipeline/
├── candidates/           ← 待处理的候选内容
│   ├── {uuid1}.json     ← 元数据
│   └── {uuid1}.md       ← Markdown 内容
├── published/            ← 已发布的内容
│   ├── {uuid2}.json
│   └── {uuid2}.md
└── archived/             ← 归档（拒绝/过期）
    ├── {uuid3}.json
    └── {uuid3}.md
```

**为什么用文件存储而不是数据库？**
```
数据库适合：结构化数据、需要复杂查询
文件适合：Markdown 内容、需要版本控制、需要直接编辑

候选内容是 Markdown 文件，用文件存储：
1. 可以直接用文本编辑器编辑
2. 可以用 Git 追踪变更
3. 不需要 ORM 序列化/反序列化
```

### 创建候选内容

```typescript
export async function createCandidate(input: CandidateInput): Promise<CandidateMeta> {
  const id = randomUUID()
  const now = new Date().toISOString()
  
  const meta: CandidateMeta = {
    id,
    title: input.title,
    source: 'manual',
    status: 'draft',
    tags: input.tags || [],
    description: input.description,
    createdAt: now,
    updatedAt: now,
  }
  
  // 写入元数据
  await writeFile(
    join(CANDIDATES_DIR, `${id}.json`),
    JSON.stringify(meta, null, 2)
  )
  
  // 写入内容
  await writeFile(
    join(CANDIDATES_DIR, `${id}.md`),
    input.content
  )
  
  return meta
}
```

### 批准并发布

```typescript
export async function approveCandidate(
  id: string,
  target: 'blog' | 'portfolio',
): Promise<void> {
  const metaPath = join(CANDIDATES_DIR, `${id}.json`)
  const contentPath = join(CANDIDATES_DIR, `${id}.md`)
  
  // 读取候选内容
  const meta = JSON.parse(await readFile(metaPath, 'utf-8'))
  const content = await readFile(contentPath, 'utf-8')
  
  // 更新状态
  meta.status = 'approved'
  meta.updatedAt = new Date().toISOString()
  
  // 移动到 published 目录
  await rename(metaPath, join(PUBLISHED_DIR, `${id}.json`))
  await rename(contentPath, join(PUBLISHED_DIR, `${id}.md`))
  
  // 如果目标是博客，还需要创建 Nuxt Content 文章
  if (target === 'blog') {
    await createBlogPost(meta, content)
  }
}
```

**关键点：`rename` vs `copy + delete`**
```typescript
// 方式 1：rename（原子操作）
await rename(oldPath, newPath)
// 优点：原子性，要么成功要么失败，不会出现中间状态
// 缺点：只能在同一文件系统内

// 方式 2：copy + delete
await copyFile(oldPath, newPath)
await unlink(oldPath)
// 优点：可以跨文件系统
// 缺点：不是原子的，可能 copy 成功但 delete 失败

本项目用 rename：因为所有目录都在同一个文件系统内
```

## 🔍 对比与选型

### 内容管理方案对比

| 方案 | 优点 | 缺点 | 适合 |
|------|------|------|------|
| 文件系统 | 简单、可 Git 追踪 | 不支持复杂查询 | 小项目、Markdown 内容 |
| 数据库 | 支持查询、事务 | 需要 ORM | 结构化数据 |
| Headless CMS | 功能全、API 友好 | 有学习成本 | 中大型项目 |
| Git-based CMS | 版本控制、审查流程 | 技术门槛 | 技术博客 |

**为什么选文件系统：** 个人网站内容量小，Markdown 文件可以直接编辑和 Git 追踪，不需要复杂查询。

### 流水线模式对比

| 模式 | 原理 | 优点 | 缺点 | 适合 |
|------|------|------|------|------|
| 简单状态机 | 状态 + 转换规则 | 简单直观 | 不够灵活 | 本项目 |
| 事件驱动 | 发布事件、订阅处理 | 解耦、可扩展 | 复杂 | 微服务 |
| 工作流引擎 | 可视化流程定义 | 灵活 | 重量级 | 企业流程 |
| Git 分支 | PR + Review | 天然审核 | 技术门槛 | 技术团队 |

**为什么选简单状态机：** 内容流转只有几个固定状态（草稿→待审→批准/拒绝），不需要复杂的工作流引擎。

### AI 内容生成方案对比

| 方案 | 优点 | 缺点 | 适合 |
|------|------|------|------|
| 直接发布 | 最快 | 无审核、可能出错 | 绝对不能用 |
| 候选 + 人工审核 | 安全 | 需要人工 | 本项目 |
| AI 自审 | 自动化 | AI 可能漏判 | 对质量要求不高 |
| 多轮审核 | 最安全 | 流程长 | 企业发布 |

**本项目选择候选 + 人工审核：** AGENTS.md 明确规定"AI 只能生成候选内容，不能默认直接公开发布"。
