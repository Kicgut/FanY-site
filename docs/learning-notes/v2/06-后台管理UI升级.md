## 📖 核心概念：CRUD 界面

**类比：** 后台管理就像 Excel 表格——增删改查是基本操作，筛选排序是进阶功能。

### CRUD 是什么？

```
C = Create（创建）  → 新增用户/照片
R = Read（读取）    → 查看列表/详情
U = Update（更新）  → 修改信息/状态
D = Delete（删除）  → 移除记录

几乎所有后台管理页面都是 CRUD 的变体
```

## 💻 代码解读

### 用户管理页面

```vue
<!-- pages/admin/users.vue -->
<script setup lang="ts">
// 用户列表
const users = ref([])
const loading = ref(false)

// 筛选条件
const filters = ref({
  role: '',      // admin / friend / viewer
  status: '',    // active / disabled
  aiAccess: '',  // true / false
})

// 获取用户列表
async function fetchUsers() {
  loading.value = true
  const { data } = await useFetch('/api/admin/users', {
    headers: { Authorization: `Bearer ${token}` },
    query: filters.value  // 筛选条件作为查询参数
  })
  users.value = data.value?.users || []
  loading.value = false
}

// 更新用户
async function updateUser(userId: number, updates: any) {
  await $fetch(`/api/admin/users/${userId}`, {
    method: 'PATCH',
    headers: { Authorization: `Bearer ${token}` },
    body: updates
  })
  await fetchUsers()
  ElMessage.success('更新成功')
}
</script>

<template>
  <el-table :data="users" v-loading="loading">
    <el-table-column prop="username" label="用户名" />
    <el-table-column prop="role" label="角色">
      <template #default="{ row }">
        <el-tag :type="row.role === 'admin' ? 'danger' : 'info'">
          {{ row.role }}
        </el-tag>
      </template>
    </el-table-column>
    <el-table-column prop="status" label="状态">
      <template #default="{ row }">
        <el-switch
          v-model="row.status"
          active-value="active"
          inactive-value="disabled"
          @change="updateUser(row.id, { status: row.status })"
        />
      </template>
    </el-table-column>
  </el-table>
</template>
```

**关键点：`#default="{ row }"` 是什么？**
```
这是 Vue 的作用域插槽（Scoped Slot）

<el-table-column>
  <template #default="{ row }">
    <!-- row 是当前行的数据 -->
    {{ row.username }}
  </template>
</el-table-column>

类比：Excel 的每一行是一个 row，列头是 prop
El-Table 把每一行的数据通过插槽传给你，你可以自定义渲染
```

### 存储状态面板

```vue
<!-- pages/admin/storage.vue -->
<script setup lang="ts">
const stats = ref({
  ecs: { used: 0, total: 0, photos: 0 },
  local: { used: 0, total: 0, photos: 0 },
})

// 计算使用百分比
const ecsPercent = computed(() => 
  stats.value.ecs.total > 0 
    ? Math.round(stats.value.ecs.used / stats.value.ecs.total * 100) 
    : 0
)

// 颜色判断
const ecsColor = computed(() => {
  if (ecsPercent.value >= 90) return '#F56C6C'  // 红色
  if (ecsPercent.value >= 70) return '#E6A23C'  // 黄色
  return '#67C23A'  // 绿色
})
</script>

<template>
  <el-card>
    <template #header>ECS 存储</template>
    <el-progress 
      :percentage="ecsPercent" 
      :color="ecsColor"
      :format="p => `${p}% (${stats.ecs.used}MB / ${stats.ecs.total}MB)`"
    />
  </el-card>
</template>
```

**关键点：`computed` 是什么？**
```
computed = 计算属性，会自动追踪依赖并缓存结果

const ecsPercent = computed(() => 
  stats.value.ecs.used / stats.value.ecs.total * 100
)

当 stats 变化时，ecsPercent 自动重新计算
当 stats 没变时，ecsPercent 返回缓存值（不会重复计算）

类比：Excel 公式 =A1/B1*100
  A1 或 B1 变了，公式结果自动更新
  A1 和 B1 没变，显示之前的值
```

## 🔍 对比与选型

### UI 组件库对比（Vue 生态）

| 组件库 | 风格 | 组件数 | 优点 | 缺点 |
|--------|------|--------|------|------|
| Element Plus | 企业风 | 70+ | 组件全、文档好 | 包体积大 |
| Ant Design Vue | 企业风 | 60+ | 设计规范 | 定制困难 |
| Naive UI | 现代风 | 80+ | TypeScript 优先 | 生态小 |
| Vuetify | Material | 80+ | 设计一致 | 学习曲线 |

**为什么选 Element Plus：** Vue 3 生态最成熟的企业级组件库，组件齐全（表格、表单、弹窗、进度条），中文文档完善。

### 后台框架对比

| 框架 | 技术栈 | 优点 | 缺点 | 适合 |
|------|--------|------|------|------|
| Vue-Element-Admin | Vue + Element | 开箱即用 | Vue 2、维护减少 | 快速搭建 |
| Ant Design Pro Vue | Vue + Ant Design | 功能全 | 体积大 | 企业项目 |
| Vben Admin | Vue 3 + Naive UI | 现代 | 学习成本 | 新项目 |
| 自己搭 | Nuxt + Element Plus | 完全可控 | 需要时间 | 学习项目 |

**为什么自己搭：** 这是学习项目，自己搭建后台能深入理解每个组件的用法，而不是用现成框架黑盒。

### 数据展示方式对比

| 方式 | 适合 | 优点 | 缺点 |
|------|------|------|------|
| 表格 | 多条结构化数据 | 信息密度高 | 移动端不友好 |
| 卡片 | 图片/内容展示 | 视觉友好 | 信息密度低 |
| 列表 | 简单项 | 轻量 | 功能有限 |
| 树形 | 层级数据 | 结构清晰 | 实现复杂 |

**用户管理用表格：** 用户数据是结构化的（姓名、角色、状态），表格最合适。
**照片管理用卡片：** 照片需要预览，卡片 + 缩略图最合适。
