## 📖 核心概念：集成测试

**类比：** 集成测试就像期末考试——不是考单个知识点，而是考你能不能把所有知识串起来用。

### 测试金字塔

```
        /\
       /  \      E2E 测试（端到端）
      /    \     - 模拟真实用户操作
     /------\    - 最慢、最贵、最少
    /        \
   / 集成测试  \  - 测试模块之间的协作
  /------------\  - 中等速度和成本
 /   单元测试    \ - 测试单个函数/组件
/________________\ - 最快、最便宜、最多
```

**本项目的测试：**
```
单元测试：权限函数、CIDR 匹配、状态转换
集成测试：API 端点、数据库操作、文件存储
E2E 测试：浏览器访问、登录、上传、审核
```

### Docker 构建

```bash
# 在 ECS 上构建
docker build -t personal-website:v2 .

# 构建过程：
# 1. FROM node:22-alpine        ← 基础镜像
# 2. RUN corepack prepare pnpm  ← 安装 pnpm
# 3. COPY package.json ...      ← 复制依赖文件
# 4. RUN pnpm install           ← 安装依赖
# 5. COPY . .                   ← 复制源码
# 6. RUN npx nuxi build         ← 构建
# 7. RUN pnpm prune --prod      ← 删除开发依赖

# 构建成功后：
# personal-website:v2   685288cc260e   3.27GB   732MB
```

### API 端点验证

```bash
# 照片 API（公开）
curl -s http://localhost:3000/api/photos
# {"success":true,"data":{"photos":[],"total":0}} ✅

# 审核 API（需要 token）
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/api/admin/photos/pending
# 401（未认证，正常） ✅

# 后台页面（SSR）
curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/admin/users
# 200 ✅
```

**关键点：为什么 401 是"正常"？**
```
审核 API 需要认证，未登录访问应该返回 401

如果返回 200 → 说明权限检查有问题（任何人都能访问）
如果返回 401 → 说明权限检查正常工作 ✅
如果返回 500 → 说明服务器内部错误

测试时要验证：
- 未登录 → 401
- 登录但无权限 → 403
- 登录且有权限 → 200
```

## 🔍 对比与选型

### 部署方式对比

| 方式 | 原理 | 优点 | 缺点 | 适合 |
|------|------|------|------|------|
| SSH + 手动 | 人肉操作 | 简单 | 慢、易出错 | 一次性 |
| Docker + 脚本 | 容器化 + 自动化 | 可重复、可靠 | 需要写脚本 | 本项目 |
| CI/CD | 自动构建部署 | 全自动 | 配置复杂 | 团队项目 |
| K8s | 容器编排 | 高可用、自动扩缩 | 重量级 | 大规模 |

**本项目选择 Docker + 脚本：** 单服务器部署，Docker 保证环境一致，脚本自动化减少人为错误。

### 回滚策略对比

| 策略 | 原理 | 优点 | 缺点 | 适合 |
|------|------|------|------|------|
| 镜像版本 | docker run old-image | 秒级回滚 | 需要保留旧镜像 | Docker |
| 数据库备份 | 恢复备份 | 数据也回滚 | 丢失新数据 | 数据变更 |
| Git revert | 重新构建部署 | 代码也回滚 | 需要重新构建 | 代码问题 |
| 蓝绿部署 | 两套环境切换 | 零停机 | 需要双倍资源 | 高可用 |

**本项目选择镜像版本：** Docker 镜像标签（v1、v2）天然支持版本管理，回滚只需 `docker run v1`。

### 监控方案对比

| 方案 | 功能 | 复杂度 | 成本 | 适合 |
|------|------|--------|------|------|
| curl 手动检查 | 最基础 | 最低 | 无 | 开发测试 |
| 脚本定时检查 | 自动化 | 低 | 无 | 小项目 |
| UptimeRobot | 外部监控 | 中 | 免费/付费 | 推荐 |
| Prometheus + Grafana | 全面监控 | 高 | 免费 | 中大型 |

**本项目建议：** 先用脚本定时检查，后续可加 UptimeRobot 外部监控（免费版支持 50 个监控）。
