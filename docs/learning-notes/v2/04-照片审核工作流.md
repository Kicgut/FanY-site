## 📖 核心概念：状态机

**类比：** 照片的生命周期就像快递包裹——下单→发货→运输→签收，每一步都有明确的状态。

### 照片审核状态机

```
                    ┌──────────────┐
                    │   上传完成    │
                    └──────┬───────┘
                           │
                           ▼
                    ┌──────────────┐
         ┌────────│  pending      │ ← 等待审核
         │        │  (待审核)      │
         │        └──────┬───────┘
         │               │
    ┌────┴─────┐    ┌────┴─────┐
    │ rejected │    │ approved │
    │ (拒绝)   │    │ (批准)   │
    └──────────┘    └────┬─────┘
                         │
                         ▼
                  ┌──────────────┐
                  │   active     │ ← 出现在公开页面
                  │  (已发布)    │
                  └──────────────┘
```

### 为什么需要审核？

```
场景：你的朋友上传了一张照片

没有审核：
  朋友上传 → 直接公开 → 可能有不合适的内容 → 你尴尬

有审核：
  朋友上传 → 待审核 → 你检查 → 合适就批准，不合适就拒绝
```

## 💻 代码解读

### 审核 API

```typescript
// server/api/admin/review/photos/[id].post.ts
export default defineEventHandler(async (event) => {
  const user = event.context.authUser
  if (!user || !requireAdmin(user)) {
    throw createError({ statusCode: 403, message: '需要管理员权限' })
  }

  const photoId = getRouterParam(event, 'id')
  const body = await readBody(event)
  
  // action: 'approve' | 'reject'
  const { action, visibleTo } = body

  if (action === 'approve') {
    // 批准：更新状态为 approved + active
    await prisma.photo.update({
      where: { id: Number(photoId) },
      data: {
        reviewStatus: 'approved',
        status: 'active',
        visibleTo: visibleTo ? JSON.stringify(visibleTo) : null,
        reviewedAt: new Date(),
      }
    })
    
    // 记录审计日志
    await createAuditLog(user.id, 'approve_photo', `photo:${photoId}`)
  }
  
  if (action === 'reject') {
    await prisma.photo.update({
      where: { id: Number(photoId) },
      data: {
        reviewStatus: 'rejected',
        status: 'archived',
        reviewedAt: new Date(),
      }
    })
    
    await createAuditLog(user.id, 'reject_photo', `photo:${photoId}`)
  }
})
```

**关键点：`getRouterParam` 是什么？**
```
路由：/api/admin/review/photos/123
                                 ↑ 这是动态参数

Nuxt 文件命名：server/api/admin/review/photos/[id].post.ts
                                      ↑ [id] 匹配 URL 中的 123

getRouterParam(event, 'id') → 返回 '123'
```

### 审核中间件

```typescript
// server/middleware/review-gate.ts
// 拦截公开页面请求，确保待审核内容不被展示

export default defineEventHandler(async (event) => {
  const path = getRequestURL(event).pathname
  
  // 只拦截公开页面和 API
  if (path.startsWith('/admin') || path.startsWith('/api/admin')) return
  
  // 公开页面查询照片时，自动过滤未审核的
  // 这个逻辑在 API 层实现，中间件做额外保护
})
```

**为什么需要中间件 + API 层双重保护？**
```
API 层：查询时过滤 reviewStatus = 'approved'
中间件层：兜底保护，防止遗漏

类比：家门有两道锁
  - 第一道：大门锁（API 过滤）
  - 第二道：房间锁（中间件保护）
  - 单独一道锁可能被绕过，两道更安全
```

### 前端审核页面

```vue
<!-- pages/admin/review.vue -->
<script setup lang="ts">
const pendingPhotos = ref([])
const loading = ref(false)

// 获取待审核照片
async function fetchPending() {
  loading.value = true
  const { data } = await useFetch('/api/admin/photos/pending', {
    headers: { Authorization: `Bearer ${token}` }
  })
  pendingPhotos.value = data.value?.photos || []
  loading.value = false
}

// 审核操作
async function handleReview(photoId: number, action: 'approve' | 'reject') {
  await $fetch(`/api/admin/review/photos/${photoId}`, {
    method: 'POST',
    headers: { Authorization: `Bearer ${token}` },
    body: { action }
  })
  
  // 刷新列表
  await fetchPending()
  
  ElMessage.success(action === 'approve' ? '已批准' : '已拒绝')
}
</script>
```

## 🔍 对比与选型

### 审核模式对比

| 模式 | 原理 | 优点 | 缺点 | 适合 |
|------|------|------|------|------|
| 人工审核 | 管理员手动检查 | 最安全 | 耗时 | 个人网站、小团队 |
| AI 预审 + 人工复核 | 先用 AI 过滤明显违规 | 效率高 | AI 可能误判 | 中型社区 |
| 纯 AI 审核 | 全自动 | 最快 | 误判率高 | 大平台 |
| 社区举报 | 用户举报后人工处理 | 成本低 | 滞后 | UGC 平台 |

**为什么选人工审核：** 个人网站照片量少，人工审核最可靠，不会误删重要内容。

### 状态管理方式对比

| 方式 | 示例 | 优点 | 缺点 |
|------|------|------|------|
| 数据库字段 | status='approved' | 简单、可查询 | 不够灵活 |
| 状态机库 | XState | 可视化、类型安全 | 学习成本高 |
| 事件溯源 | EventStore | 可追溯、可回放 | 复杂度高 |
| 枚举 + 校验 | as const + 函数检查 | 轻量、够用 | 需要手动维护 |

**本项目选择：数据库字段 + as const 常量**
```
简单够用，不需要引入额外依赖
状态转换通过代码逻辑控制，不依赖第三方库
```
