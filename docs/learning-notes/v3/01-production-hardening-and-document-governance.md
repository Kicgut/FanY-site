# Phase 13：生产加固、运维闭环与文档治理

> 这是学习记录，不替代当前架构和运行手册。当前事实以 `docs/architecture/`、`docs/operations/` 为准。

## 学到的内容

### 1. 二次验证需要完整的用户体验

TOTP 不能只显示密钥文本；服务端生成 `otpauth` URI 后，前端应生成二维码，同时保留可复制密钥和恢复码。恢复码再生成必须使旧恢复码失效。

### 2. 后台异步工作必须可观察

内容导入和缩略图重建都应表示为 Job，而不是隐藏在请求或脚本里。任务至少要能看到状态、错误、执行结果，并支持安全的重试或取消。

### 3. 存储操作需要先检查再修复

缺失原图、缺失缩略图和孤儿文件要先通过一致性扫描报告，再通过受审计的修复任务处理。文件路径必须受服务端白名单限制。

### 4. 生产验证要与部署绑定

构建通过不等于上线可用。发布流程至少包含迁移、备份、健康检查、未认证安全边界检查、回滚镜像保留和恢复演练。

### 5. 文档必须区分事实与过程

当前架构、设计和运行手册应保持少而准；已完成任务不应继续占据导航。学习过程可以保留在版本化学习笔记中，架构历史只保留总体快照与关键变化。
