# 架构历史 v1.0

## 总体架构

初始版本以 Nuxt 个人网站为核心：公开博客、作品集、照片展示与后台管理运行在单一应用中，SQLite/Prisma 保存业务数据，ECS 提供线上运行环境。

照片与备份需求引入本地服务器：ECS 负责站点和缩略图，本地服务器保存原图与备份。该阶段确定了容器化部署、Nginx 反向代理和后续 FRP 内部连通的方向。

## 关键变化

- 建立 Nuxt、Nitro、Prisma 与 SQLite 的基础边界。
- 确定公开内容、后台管理和照片资产是同一站点的核心域。
- 确定 ECS 与本地资源分工，但尚未形成后续的受控 API、审计与 Jobs 运维体系。
