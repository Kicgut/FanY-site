## 版本规则

- 小变更直接更新当前文档底部的“架构更新记录”，按 `v主.次.补丁` 递增。
- 跨模块、数据流、权限模型或部署拓扑的大变更，先新增 `plan-architecture.md` 设计；实施完成后合并到当前文档并删除该计划文档。
- 新的大版本出现时，将上一版本的总体摘要写入 `history/vX.0.md`，再将当前文档提升为新的主版本。
