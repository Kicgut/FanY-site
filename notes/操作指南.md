# Ubuntu 本地高权限操作指南

## 结论

Ubuntu 上**不需要、也不应该访问一个单独的“本地后台网址”**。网站后台仍然运行在 ECS，正常公网地址是：

```text
http://120.26.231.150/admin/login
```

但是直接访问这个公网地址会被识别为远程访问，管理员角色仍然不能执行本地高风险操作。要获得 `local_trusted`，需要从 Ubuntu 建立 SSH 本地端口转发，让浏览器访问 ECS 的本机回环地址。

## 推荐方式：Ubuntu 建立 SSH 隧道

在 Ubuntu 终端执行：

```bash
ssh -N -T \
  -o ExitOnForwardFailure=yes \
  -L 18080:127.0.0.1:80 \
  root@120.26.231.150
```

保持这个终端运行，然后在 Ubuntu 的浏览器访问：

```text
http://127.0.0.1:18080/admin/login
```

登录管理员账号后访问：

```text
http://127.0.0.1:18080/admin/local-ops
```

这里的 `127.0.0.1:18080` 是 Ubuntu 本地 SSH 隧道，不是公网端口。ECS 收到请求时会将其识别为本机可信来源。

如果 ECS 的 SSH 登录用户不是 `root`，将命令中的 `root` 替换为实际用户。若 Ubuntu 没有 SSH 私钥或没有 ECS 登录权限，应先配置 SSH key，不能把密码写入脚本或仓库。

## 如何确认已经获得本地最高权限

登录后检查以下几点：

1. `/admin` 侧栏出现“本地高权限”菜单。
2. `/admin/local-ops` 页面显示 `Local Trusted` 或“本地可信来源”。
3. `/admin/hermes/skills` 页面显示“Sync Skills”按钮，而不是“远程只读”。
4. 页面中的高风险操作仍需二次确认；本地可信来源不等于跳过审计。

也可以在 Ubuntu 上验证来源接口：

```bash
curl -s http://127.0.0.1:18080/api/admin/access-origin
```

返回数据中的 `origin` 必须是：

```json
"local_trusted"
```

## Skill 同步

只有通过上述 SSH 隧道进入后台后，才在以下页面点击同步：

```text
http://127.0.0.1:18080/admin/hermes/skills
```

直接访问 `http://120.26.231.150/admin/hermes/skills` 时，即使账号是管理员，也只应当看到远程只读提示，不能同步 Skill。

## Ubuntu 上手动执行照片回流

本地高权限网页操作和照片回流服务是两条不同链路。需要立即处理回流任务时，在 Ubuntu 终端执行：

```bash
systemctl --user start photo-backflow.service
systemctl --user status photo-backflow.service --no-pager
```

查看最近日志：

```bash
journalctl --user -u photo-backflow.service -n 80 --no-pager
```

自动回流定时器当前约每 5 分钟执行一次：

```bash
systemctl --user is-active photo-backflow.timer
systemctl --user list-timers photo-backflow.timer
```

## 不要使用的方式

- 不要把 `120.26.231.150/admin` 当成本地高权限入口。
- 不要通过修改浏览器请求头伪造 `local_trusted`。
- 不要把 Ubuntu 公网 IP 直接加入可信 CIDR 作为长期方案。
- 不要为了同步 Skill 在 ECS 上直接执行 shell 或修改容器源码。
- 不要把 SSH 私钥、FRP token、JWT secret 或真实 `.env` 写入 Git。

## 隧道关闭

使用完毕后，在运行 SSH 隧道的终端按 `Ctrl+C` 关闭。关闭后，公网地址仍可访问普通后台，但会恢复为远程权限，不能执行本地高风险操作。
